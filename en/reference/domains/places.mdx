Places are physical venues users discover and attach to events. Media, reviews, and reactions are nested under places.

## Table: `places`

Key fields: `name`, `description`, `categoryID`, geo coordinates, pricing, `createdBy`, soft-delete via `isDeleted`.

## API endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/places` | List places |
| POST | `/places` | Create (multipart for images) |
| GET | `/places/:id` | Get by ID |
| PUT | `/places/:id` | Update |
| DELETE | `/places/:id` | Soft delete |

Nested routes:

- `GET/POST /places/:placeID/reviews`
- `GET/POST /places/:placeID/media`

OpenAPI: **API Reference** tab → Places tag.
